- cube representation & cube turns

p2: Scrambler
- random set of moves (avoid invalid sequences)
- apply scrambler to cube

p3: custom state
- get array for cube with a custom user input

p4: Solver 1 (kociemba library)
- gets a array for cube representation and solves it

p5: move to website
- Frontend: react with threejs
- Backend: fastAPI

p6: Scan cube with camera
- open cam and scans the 6 side

p7: Solver 2 (thistlethwaite's algorithm or cfop)
- Hard code algorithm for solving

p8: Solver 3 (kociemba algorithm)

Testing done after each step
